# المتطلبات الأولية لمشروع "بصير"

## المتطلبات الوظيفية

- منصة موحدة تدعم وضع الأفراد ووضع الأعمال.
  - **وضع الأفراد:** إدارة الميزانيات، كشف الاشتراكات، نصائح مالية مخصصة.
  - **وضع الأعمال:** نظام فواتير إلكترونية، إدارة العملاء والموردين، إدارة المخزون.
- ربط آمن للحسابات البنكية (Open Banking) مع دعم البنوك المحلية والدولية.
- تصنيف تلقائي للمعاملات والنفقات باستخدام الذكاء الاصطناعي.
- إدارة الميزانيات والأهداف المالية مع تقارير تحليلية.
- نظام فواتير إلكترونية متوافق مع الأنظمة المحلية (مثل هيئة الزكاة والضريبة والجمارك).
- إدارة العملاء والموردين (CRM Lite) مع سجل تاريخي للمعاملات.
- إدارة المخزون البسيطة مع تنبيهات انخفاض المخزون.
- مسح ضوئي للإيصالات (OCR) واستخلاص البيانات بدقة عالية.
- تقارير مالية وضريبية مبسطة مع دعم التصدير إلى Excel وPDF.
- دعم العملات المتعددة وتحديث أسعار الصرف بشكل لحظي.

### أمثلة عملية

- **وضع الأفراد:**
  - مثال: يمكن للمستخدم تحديد هدف ادخار شهري، وسيقوم النظام بتقديم نصائح لتحقيق الهدف.
- **وضع الأعمال:**
  - مثال: يمكن للشركات الصغيرة إصدار فواتير إلكترونية متوافقة مع الأنظمة المحلية بسهولة.

## المتطلبات غير الوظيفية

- أمان من الدرجة المصرفية (تشفير شامل، E2EE) مع مصادقة متعددة العوامل (MFA).
- أداء عالي وقابلية التوسع باستخدام بنية Microservices وتقنيات Cloud Native.
- دعم اللغة العربية بشكل كامل في جميع الواجهات والخدمات، مع تجربة مستخدم متميزة.
- الامتثال للمعايير المحلية والدولية (ISO 27001, SOC 2, PDPL, GDPR).
- تجربة مستخدم سلسة وسهلة على جميع الأجهزة (Responsive Design).
- وقت استجابة أقل من 2 ثانية لمعظم العمليات.

### تحقيق الامتثال

- **ISO 27001:** تنفيذ سياسات أمان شاملة وإجراء اختبارات دورية.
- **GDPR:** ضمان حماية بيانات المستخدمين وتوفير خيارات لإدارة الخصوصية.

## القيود والافتراضات

- ضرورة التكامل مع البنوك المحلية المعتمدة لضمان الربط البنكي.
- الالتزام بالتحديثات التنظيمية المستمرة في السوق المستهدف.
- توفر فريق هندسي متخصص في الذكاء الاصطناعي والتقنيات السحابية.
- الاعتماد على خدمات سحابية موثوقة (مثل AWS أو Google Cloud) لضمان الاستقرار.
- قيود زمنية: يجب إطلاق النموذج الأولي (MVP) خلال 6 أشهر.

## الأولويات

- **Critical:**
  - الأمان والامتثال للمعايير.
  - نظام الفواتير الإلكترونية.
  - الربط البنكي.

- **High:**
  - إدارة الميزانيات والأهداف.
  - دعم اللغة العربية.

- **Medium:**
  - إدارة المخزون.
  - تقارير تحليلية متقدمة.

- **Low:**
  - دعم العملات المتعددة.
  - تخصيص واجهات المستخدم.

## تحليل المخاطر

- **مخاطر الأمان:** احتمالية حدوث اختراقات.
  - **خطة التخفيف:** تنفيذ اختبارات اختراق دورية وتحديثات أمنية مستمرة.
- **مخاطر الأداء:** تأخير في استجابة النظام.
  - **خطة التخفيف:** تحسين البنية التحتية وإجراء اختبارات أداء دورية.
- **مخاطر الامتثال:** تغييرات مفاجئة في اللوائح التنظيمية.
  - **خطة التخفيف:** متابعة مستمرة للتحديثات التنظيمية.

## التكامل مع الوثائق الأخرى

- ربط المتطلبات بخارطة الطريق لضمان تحقيق الأولويات في الوقت المحدد.
- ضمان توافق المتطلبات مع خطة الجودة وخطة الأمان.
- استخدام المتطلبات كأساس لتطوير قصص المستخدم وخطة العمل.
- **مثال:** يتم استخدام متطلبات الربط البنكي لتطوير قصة مستخدم تتعلق بإدارة الحسابات البنكية.
